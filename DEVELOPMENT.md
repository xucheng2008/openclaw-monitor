## ⏰ 进度反馈机制

### 1. 任务完成反馈（立即）

**触发条件**: 子 Agent 完成任务并提交代码

**反馈内容**:
- ✅ 任务完成通知
- 📝 提交摘要（文件变更、功能点）
- 🔗 GitHub Issue 链接
- 📊 更新后的整体进度

**流程**:
```
子 Agent 完成 → boss 验收 → 提交代码 → 更新 Issue → 发送完成通知
```

### 2. 定时进度报告（每小时）

**频率**: 每小时整点

**内容**: 
- 各任务当前状态
- 过去 1 小时的提交记录
- 已完成的任务
- 遇到的阻塞问题
- 下一步计划
- 整体进度百分比
